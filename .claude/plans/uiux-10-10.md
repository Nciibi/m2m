# UI/UX Upgrade: 6.5 → 10/10

**Goal**: Transform the functional-but-basic interface into a polished, professional desktop app
with state management, accessibility, responsive design, micro-animations, and developer tooling.

**Timeline**: 2 weeks / 6 phases, each independently shippable.

```tsx
const [theme, setTheme] = useState<'dark' | 'light'>('dark');
useEffect(() => {
  document.documentElement.setAttribute('data-theme', theme);
}, [theme]);
// System preference detection:
useEffect(() => {
  const mq = window.matchMedia('(prefers-color-scheme: light)');
  setTheme(mq.matches ? 'light' : 'dark');
  mq.addEventListener('change', e => setTheme(e.matches ? 'light' : 'dark'));
}, []);
```

```typescript
  const [loading, setLoading] = useState<Record<string, boolean>>({});
  const [theme, setTheme] = useState<'dark' | 'light'>('dark');

  // All event listeners bound once
  // All IPC command wrappers
  return { ... };
}
```

**New file: `src/components/AppShell.tsx`** — Layout wrapper:
```typescript
// Handles:
// - Keyboard shortcuts (global)
// - Theme detection
// - Toast container
// - Auto-scroll management
```

**Modified: `src/App.tsx`** — Slim down to 40 lines:
```typescript
function App() {
  const state = useM2MState();
  return (
    <AppShell theme={state.theme} toasts={state.toasts}>
      {state.view === 'setup' && <SetupView />}
      {state.view === 'vault' && <VaultView onUnlock={state.unlockVault} />}
      {state.view === 'hub' && <HubView {...state.hubProps} />}
      {state.view === 'chat' && <ChatView {...state.chatProps} />}
      {state.view === 'settings' && <SettingsView {...state.settingsProps} />}
    </AppShell>
  );
}
```

**Modified: Every view** — Reduce props from 30+ to 5–8 by using context/hooks internally.

### Impact
| Before | After |
|--------|-------|
| 650-line App.tsx | 40-line router |
| 40 inline state vars | Encapsulated in hook |
| Views with 30+ props | Views with 5–8 props |
| No reuse between views | Hook pattern for cross-view state |

## Phase 3 — Animations & Micro-Interactions (Day 9–10)

### What to add

| Element | Animation | CSS technique |
|---------|-----------|---------------|
| App mount | Glass card fade-in + scale | `animation: appFadeIn` (exists, keep) |
| View transitions | Slide left/right between hub↔chat | CSS `translateX` with `will-change` |
| Message send | Bubble slides up from input | `animation: msgSlide` (exists, keep) |
| New message (received) | Bubble slides in + subtle glow pulse | `translateY(10px)` → `translateY(0)` + box-shadow pulse |
| Connection state change | Status badge color transition + ring | `background-color` + `box-shadow` transitions |
| Modal open | Backdrop fade-in + content scale-up | Existing `modalFadeIn` + `modalZoomIn` |
| Button hover | Shine sweep (exists) + subtle scale | `scale(1.02)` on hover |
| Invite generation | Copy button shows "Copied!" feedback for 2s | State toggle, `animation: pop` |
| File transfer progress | Animated progress bar with gradient sweep | `@keyframes shimmer` on the bar fill |
| Scrollbar | Thin, themed, fades when idle | Auto-hide after 2s of no scroll |
| Toast | Slide in from right, progress bar counts down | Existing + `animation: progress-shrink` |
| Typing indicator | Three dots bounce with staggered delay | Existing `dotBounce` pattern |
| Error state | Shake animation on invalid input | `@keyframes shake` on `.vault-error` |
| Vault unlock | Lock icon animates open on success | `scale` + `rotate` on the lock icon |

### Performance notes
- All animations use `transform` and `opacity` only (GPU-composited, no layout triggers)
- Respect `prefers-reduced-motion`: wrap all animations in `@media (prefers-reduced-motion: no-preference)`
- `will-change: transform` only on elements that animate (not on every hover target)

## Phase 4 — Accessibility & Responsive (Day 11–12)

### 4a — Accessibility (a11y)

| Requirement | Implementation |
|-------------|---------------|
| Focus management | Tab trapping in modals, auto-focus first input, focus ring on all interactive elements |
| ARIA labels | Every icon-only button gets `aria-label`, every input gets `aria-labelledby` |
| Role attributes | `role="dialog"` on modals, `role="tablist"` on hub tabs, `role="tab"` on each tab |
| Live regions | `aria-live="polite"` on message list (new messages announced) |
| Alt text | All decorative images hidden from screen readers (`role="presentation"` / `aria-hidden`) |
| Tab order | Logical tab order: sidebar → content → input |
| Reduced motion | `prefers-reduced-motion` disables all non-essential animations |
| Contrast ratio | Dark mode: 7:1 minimum contrast (passes WCAG AAA on current colors). Light mode: 4.5:1 minimum |
| Color independence | Status not conveyed by color alone — add text labels to status badges |
| Keyboard navigation | Arrow keys in conversation list, Enter to open, Delete to delete (with confirmation) |

### 4b — Responsive Design

| Breakpoint | Behavior |
|------------|----------|
| > 1000px | Full desktop layout (current default) |
| 600–1000px | Tablet: reduced padding (24px → 16px), sidebar collapses to icon bar |
| < 600px | Mobile: full-bleed container, bottom tab bar, floating action button |

```css
@media (max-width: 1000px) {
  .app-container { max-width: 100%; margin: 0; border-radius: 0; }
  .header { padding: 14px 20px; }
  .messages { padding: 16px 20px; }
}

@media (max-width: 600px) {
  body { padding: 0; }
  #root { padding: 0; }
  .app-container { height: 100vh; border-radius: 0; }
  .invite-section { max-width: 100%; padding: 0 16px; }
  .card { padding: 20px; }
  .fingerprint-comparison { flex-direction: column; }
  .fp-grid { grid-template-columns: repeat(2, 1fr); }
  .conversation-item { padding: 14px; }
  .settings-row { flex-direction: column; align-items: flex-start; }
}
```
