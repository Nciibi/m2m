# Mission: Generate Interface Implementation Prompts

The UI/UX Design Bible is now the authoritative specification for this project.

Your task is **NOT** to implement the interfaces.

Your task is to generate a complete set of **implementation prompts**, one Markdown document per interface.

These prompts will later be consumed by autonomous coding agents.

Each prompt should contain every instruction required for an implementation agent to build **only that interface** while strictly following the Design Bible.

---

# Folder Structure

Create

```text
docs/ui/implementation_prompts/
```

# Every Prompt Must Include

Every Markdown file should instruct an implementation agent to build **only that interface**.

Each file must contain:

* Mission
* Scope
* Files expected to be modified
* Components to reuse
* Components to create
* Layout hierarchy
* Pixel-perfect implementation requirements
* Exact spacing rules
* Typography
* Colors
* Icons
* Hover states
* Focus states
* Active states
* Disabled states
* Loading states
* Empty states
* Error states
* Animations
* Keyboard shortcuts
* Mouse interactions
* Accessibility
* Responsive behavior
* Performance considerations
* Security considerations (where applicable)
* Edge cases
* Acceptance criteria
* Self-review checklist

---

# Important Rules

Each prompt must clearly instruct the implementation agent:

* Never redesign the interface.
* Never improvise.
* Never invent layouts.
* Follow the Design Bible exactly.
* Reuse existing components whenever possible.
* Preserve application architecture.
* Preserve functionality.
* Produce production-quality code.

---

# Acceptance Criteria

Every prompt should end with a checklist that the implementation agent must satisfy before considering the interface complete.

The prompts should be detailed enough that a coding agent can implement the interface without referring back to any other documentation except the Design Bible.
